This section contains information pertaining to the setup and usage of HLAPI.

Requirements

Minimal Requirements Required to Run HLAPI

  • The AAMotion is built in .Net Framework 4.0 so it is compatible with Windows XP and above

  • A working version of PCSuite must be installed on the computer that HLAPI will be running on

Environment

Setting up the environment

The HLAPI is written .Net Framework 4.0, therefore it will be compatible with most operating systems down to Windows XP, all example codes that will be given has been tested on Visual Studio 2017. Code examples may at times make use of constructs that are only available in C#6.0 or 7.0 such as expression bodies to represent certain types of properties or the use of string interpolation to better represent the position of variable in a string manipulated sequence

Therefore in cases where the user's version of Visual Studio Build tools do not support C# up to the afore-mention versions, please contact via the email link below for technical support

  • To get started, add a AAMotion.dll to the project references (a Visual Studio project). Thereafter, instantiate the MotionController class and connect via the communication path that you are attempting. Prior to HLAPI AAMotion v2.0.0 and below, it is necessary to download and store a local copy of this. However in future implementations, the AAMotion will be consolidated and maintained by the nuget server which will be available to the general public.

Introduction

This section will briefly go over the HLAPI AAMotion for more detailed discussions regarding the structure of the code in subsequent sections in HLAPI Structure

  • The AAMotion HLAPI talks with the motion controllers via a communication server. This server can be connected by multiple clients.

    This means that PCSuite; an application software which provides a front end GUI which can be used to configure a motion controller can also inter-operate with the AAMotion simultaneously

    The communication Server (also known as CommServer) handles all the incoming and outgoing traffic to any number of controllers

    In fact there can be multiple instances of HLAPI and PCSuite and it will all still work transparently with each other. A detailed discussion of this can also be found on the remarks section of MotionController

  • Once AAMotion DLL has been installed at programmer's operating environment, a simple Connect(String) is necessary to establish connection between user and motion controller. Thereafter, the user can use any of the Nodes (as described in more detail in HLAPI Structure). To control either a single axis motion, a vector/multiple axis motion or an IO pertaining to the relavent port that it is connected to.
