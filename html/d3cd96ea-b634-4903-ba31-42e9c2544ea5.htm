High Level API Documentation

This topic contains the following sections:

  • HLAPI Usage
  • Logging

Required introduction

  • All method calls have example code for how to call the particular methods

  • All example code will compile for Visual Studio 2017

  • All example code uses Console as a means to demonstrate each method call and its usage

Checklist

  1. First step: Check that the DLL is correct

  2. Second step: Check that PCSuite has been installed successfully

  3. Third step: Verify that communication is possible via PCSuite

  4. Fourth step: Attempting to connect via HLAPI should be successful

HLAPI Usage

Usage Procedures

  1. First step: All method calls should be checked for true or false

  2. Second step: All failed methods should then result in the code checking for either ErrorMessage or ErrorCode

  3. Third step: To start any operation, the user needs to instantiate the motionController class with the appropriate controller type

  4. Fourth step: Once instantiation has been complete, then call Connect Method.

  5. Fifth step: Any errors that are reported should be accompanied with the log file as described in the section below

Logging

The AAMotion is integrated with a log mechanism, so traces are transparently generated all the time. The location that it generates to is fixed and will reside on C:\PCSuite\Logs\Diagnostics*.log

When any issue is filed, it is recommended that the user attach this file along with the detailed information on how the issue can occur and or what the issue is this is to assist the support team with diagnosis to ensure that the bug/issue can be swiftly resolved
